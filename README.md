# 경력기술서 템플릿 / Career Profile Template

브라우저에서 바로 편집하고 PDF로 저장할 수 있는 한국어/영문 경력기술서 템플릿입니다.
별도 설치 없이 `index.html` 파일 하나만 열면 바로 사용할 수 있습니다.
우측 하단 **한 / EN** 버튼으로 한국어 ↔ 영문을 즉시 전환할 수 있습니다.

---

## 주요 기능

### 1. 편집 모드
화면 우측 하단의 **✏ 편집 모드** 버튼을 클릭하면 편집 모드가 활성화됩니다.
편집 모드에서는 모든 텍스트를 클릭하여 직접 수정할 수 있으며, 상단에 편집 툴바가 표시됩니다.
편집을 마치면 **✓ 편집 완료** 버튼을 클릭하여 편집 모드를 종료합니다.

---

### 2. 인라인 텍스트 편집
편집 모드에서 다음 항목들을 클릭하여 직접 수정할 수 있습니다.

- **헤더 영역**: 이름, 영문이름/직군, 핵심 역할 키워드, 재직 상태, 소속 회사, 포지셔닝 문구
- **태그바**: 핵심 역량 태그 텍스트
- **경력 섹션**: 회사명, 재직 기간, 업종/직책, 역할명, 기간, 업무 내용(불릿), 성과 칩
- **역량 섹션**: 역량 그룹 제목, 역량 항목
- **업무 스타일 섹션**: 업무 스타일 텍스트

> 각 항목에 마우스를 올리면 **툴팁 힌트**가 표시되어 작성 가이드를 확인할 수 있습니다.

---

### 3. 텍스트 서식 (편집 툴바)
편집 모드 상단 툴바에서 텍스트를 선택한 후 서식을 적용할 수 있습니다.

| 기능 | 단축키 |
|------|--------|
| **굵게 (Bold)** | `Ctrl+B` / `⌘+B` |
| *기울임 (Italic)* | `Ctrl+I` / `⌘+I` |
| 텍스트 색상 변경 | 툴바의 **Color** 버튼 클릭 |

색상 팔레트에서 진한, 회색, 연회색, 네이비, 초록, 딥그린, 딥블루, 앰버, 퍼플 등 9가지 색상을 선택할 수 있습니다.

---

### 4. 되돌리기 / 다시실행 (Undo · Redo)
편집 모드에서 구조 변경(항목 추가·삭제·드래그 순서 변경·색상 변경)을 **되돌리거나 다시실행**할 수 있습니다.

| 동작 | 방법 |
|------|------|
| 되돌리기 | 툴바 **↩** 버튼 또는 `Ctrl+Z` (contenteditable 바깥에서) |
| 다시실행 | 툴바 **↪** 버튼 또는 `Ctrl+Shift+Z` |

> **설계 원칙**
> - 텍스트 편집 중(`contenteditable` 포커스 상태) `Ctrl+Z`는 브라우저 기본 텍스트 undo로 동작
> - 구조 변경 히스토리는 **메모리에만 보관** (브라우저 캐시·localStorage 미사용)
> - 최대 **30단계**까지 기록 (스냅샷당 ~15-30KB, 총 ~0.5-1MB 이내)
> - 편집 모드 종료 시 히스토리 자동 해제 → 메모리 즉시 반환
> - JSON 불러오기 시 히스토리 리셋

---

### 5. URL 자동 링크 변환
편집 모드에서 텍스트 영역에 `https://` 로 시작하는 URL을 입력하면, 해당 필드를 벗어날 때 자동으로 클릭 가능한 링크로 변환됩니다.
- 연락처 블록, 포지셔닝, 업무 스타일, 불릿 등 모든 텍스트 영역 지원
- 태그, 칩, 역할/기간 등 단문 요소는 제외

---

### 6. 항목 추가 / 삭제 (전 영역 지원)

편집 모드에서 이력서의 **모든 항목을 자유롭게 추가하거나 삭제**할 수 있습니다.
삭제 시 화면 하단에 **실행취소** 토스트가 표시되어 실수를 복구할 수 있습니다.

#### Core 태그바
| 동작 | 방법 |
|------|------|
| 태그 추가 | 태그바 끝의 **+** 버튼 클릭 → 기본 파란색 태그 생성 |
| 태그 삭제 | 태그 우측의 **×** 버튼 클릭 |
| 태그 색상 변경 | 태그 **우클릭** → 파랑 / 초록 / 보라 / 주황 선택 |
| 태그 순서 변경 | 태그를 마우스로 드래그 |

#### 연락처 (헤더 우측)
| 동작 | 방법 |
|------|------|
| 항목 추가 | **+ 항목 추가** 버튼 클릭 |
| 항목 삭제 | 항목 우측의 **×** 버튼 클릭 |

#### 경력 섹션
| 동작 | 방법 |
|------|------|
| 경력 블록 추가 | 경력 섹션 하단 **+ 경력 추가** 버튼 클릭 (실행취소 가능) |
| 경력 블록 삭제 | 헤더 우측의 **✕ 경력** 버튼 클릭 (실행취소 가능) |
| Phase 추가 | 각 회사 블록 하단 **+ Phase 추가** 버튼 클릭 (실행취소 가능) |
| Phase 삭제 | Phase 헤더 우측의 **✕ Phase** 버튼 클릭 (실행취소 가능) |
| 불릿 항목 추가 | 불릿 목록 하단 **+ 항목 추가** 버튼 클릭 |
| 불릿 항목 삭제 | 항목 우측의 **✕** 버튼 클릭 |
| 성과 칩 추가 | 성과 행 우측 **+ 추가** 버튼 클릭 |
| 성과 칩 삭제 | 칩 내부의 **✕** 버튼 클릭 |
| 성과 칩 순서 변경 | 칩을 마우스로 드래그하여 좌우 위치 변경 |

#### 보유 역량 & 도구 섹션
| 동작 | 방법 |
|------|------|
| 스킬 항목 추가 | 각 그룹 하단 **+ 항목 추가** 버튼 클릭 |
| 스킬 항목 삭제 | 항목 우측의 **×** 버튼 클릭 |
| 그룹 추가 | 섹션 하단 **+ 그룹 추가** 버튼 클릭 (실행취소 가능) |
| 그룹 삭제 | 그룹 제목 좌측의 **×** 버튼 클릭 (실행취소 가능) |

#### 업무 스타일 섹션
| 동작 | 방법 |
|------|------|
| 문단 추가 | 섹션 하단 **+ 문단 추가** 버튼 클릭 |
| 문단 삭제 | 문단 내부 우측의 **×** 버튼 클릭 (실행취소 가능) |

> 모든 추가/삭제 버튼은 **편집 모드에서만 표시**됩니다. 인쇄 및 저장 시 자동으로 제거됩니다.

---

### 7. 드래그 앤 드롭 순서 변경
편집 모드에서 핸들을 드래그하여 항목 순서를 자유롭게 변경할 수 있습니다.

- **회사 블록 순서 변경**: 회사명 좌측의 드래그 핸들(`⠿`)을 드래그
- **Phase 순서 변경**: Phase 제목 좌측의 드래그 핸들을 드래그
- **태그 순서 변경**: 태그바의 태그를 마우스로 드래그하여 좌우 위치 변경
- **성과 칩 순서 변경**: 성과 행의 칩을 마우스로 드래그하여 좌우 위치 변경

---

### 8. 저장 기능

우측 하단에 3가지 저장 버튼이 제공됩니다.

#### ↓ HTML 저장
현재 경력기술서를 **편집 UI가 제거된 깨끗한 HTML 파일**로 저장합니다.
저장된 파일은 공유 또는 아카이브 용도로 사용할 수 있습니다.
파일명: `career_YYYY-MM-DD.html`

#### ↓ JSON 저장 (`Ctrl+S` / `⌘+S`)
현재 내용을 **JSON 버전 파일**로 저장합니다.
나중에 불러와 이어서 편집할 수 있습니다.
파일명: `career_YYYY-MM-DD.json`

#### ↑ 버전 불러오기
이전에 저장한 JSON 파일을 불러와 내용을 복원합니다.
미저장 변경사항이 있을 경우 확인 팝업이 표시됩니다.

---

### 9. 미저장 변경사항 보호
변경사항이 있는 상태에서 페이지를 닫으려 하면 **경고 메시지**가 표시됩니다.
상단 툴바의 **● 변경됨** 표시로 저장하지 않은 변경사항이 있는지 확인할 수 있습니다.
중요한 내용은 반드시 **↓ JSON 저장**으로 백업해 두세요.

---

### 10. 인쇄 / PDF 저장
화면 하단 또는 편집 모드 종료 후 **인쇄 / PDF 저장** 버튼을 클릭하면
브라우저 인쇄 대화상자가 열립니다. PDF로 저장하려면 프린터를 **"PDF로 저장"** 으로 선택하세요.
인쇄 시 편집 UI(버튼, 핸들 등)는 자동으로 숨겨집니다.

---

### 11. 반응형 레이아웃
640px 이하 모바일 화면에서는 레이아웃이 자동으로 조정됩니다.

- 헤더의 이름/연락처가 세로로 배치됩니다.
- 역량 그리드가 2열로 변경됩니다.

---

### 12. 한/영 언어 전환 (i18n)
우측 하단의 **한 / EN** 버튼을 클릭하면 전체 UI와 기본 템플릿 텍스트가 즉시 전환됩니다.

#### 번역 대상
| 영역 | 예시 |
|------|------|
| **UI 라벨/버튼** | 편집 모드, JSON 저장, 인쇄 / PDF 저장 등 |
| **편집 툴바** | 굵게, 기울임, 색상 등 툴팁 |
| **힌트 툴팁** | 각 항목의 `data-hint` 가이드 텍스트 |
| **토스트/알림** | 저장 완료, 삭제됨, 실행취소 등 |
| **섹션 제목** | 경력 → Experience, 보유 역량 & 도구 → Skills & Tools 등 |
| **기본 뷰 텍스트** | 이름, 회사명, Phase, 업무 bullet, 성과 chip, 스킬 항목, 업무 스타일 전체 |

#### 설계 원칙
- **사용자 편집 콘텐츠는 번역하지 않음** — 사용자가 직접 수정한 텍스트는 언어 전환 시에도 그대로 유지
- `data-i18n` (textContent), `data-i18n-html` (innerHTML), `data-i18n-title` (title 속성), `data-hint-i18n` (힌트 툴팁) 4가지 속성으로 번역 적용
- `data-section` 속성 기반 섹션 탐색 — 섹션 제목이 바뀌어도 기능 정상 동작
- 편집 모드 중에도 전환 가능

## 사용 방법

1. `index.html` 파일을 브라우저로 엽니다.
2. 우측 하단 **✏ 편집 모드** 버튼을 클릭합니다.
3. 각 항목을 클릭하여 본인의 정보를 입력합니다.
4. 필요한 항목은 **+ 버튼**으로 추가, 불필요한 항목은 **× 버튼**으로 삭제합니다.
5. 편집이 완료되면 **↓ HTML 저장** 또는 **↓ JSON 저장**으로 저장합니다.
6. **인쇄 / PDF 저장** 버튼으로 PDF를 생성합니다.

```
index.html   — 경력기술서 템플릿 (단일 파일, 외부 의존성 없음)
README.md    — 사용 가이드 (본 문서)
plan.md      — 한영 토글 기능 구현 계획
```

## Changelog

### v0.8 — 2026-03-23

#### 버그 수정 (태그·블록 관리 개선)
- **삭제 시 Undo 토스트 일관 적용** — 불릿 항목, 성과 칩, 스킬 항목, 태그 삭제 시에도 Undo 토스트가 표시되어 즉시 복구 가능 (기존에는 Phase/경력/연락처/문단만 지원)
- **새로 추가된 항목의 삭제 핸들러 일관 적용** — 동적으로 생성된 연락처, 문단, 불릿, 칩, 스킬 항목의 × 버튼에도 Undo 토스트 적용
- **이벤트 리스너 누수 수정** — 태그 색상 팝업(`openTagColorPopup`)과 색상 팔레트(`openColorPalette`)의 `closeHandler`를 전역 변수로 추적하여, 팝업 반복 열기 시 리스너가 누적되지 않도록 수정
- **툴바 mousedown 리스너 중복 등록 방지** — 편집 모드를 반복 On/Off 할 때 `.tb-btn` mousedown 핸들러가 중복 등록되던 문제 수정
- **스킬 그룹 삭제 Undo 시 + 버튼 복원** — 스킬 그룹 삭제 후 되돌리기 시 `initSkillControls()` 재실행으로 + 항목 추가 버튼도 함께 복구
- **Phase/경력 추가 시 칩 드래그 누락 수정** — `createNewPhase()` / `createNewCompany()` 후 `initChipDrag()` 호출 추가로 새 칩도 즉시 드래그 가능
- **HTML 내보내기 시 내부 data 속성 정리** — `getCleanHTMLString()`에서 `data-color-ready`, `data-drag-ready`, `data-add-ready`, `data-chip-drag-ready`, `spellcheck` 속성 제거 추가 (기존에는 `getPageContentSnapshot()`에서만 정리)

### v0.4 — 2026-03-21

#### 버그 수정
- **Core 태그 신규 추가 후 텍스트 편집 불가 수정** — `+` 버튼으로 태그 추가 시 `contenteditable` 속성이 설정되지 않아 텍스트 편집이 안 되던 버그 수정. 태그 추가와 동시에 `contenteditable='true'` 설정 및 자동 포커스/전체선택 적용
- **텍스트 색상 팔레트 미적용 수정** — 툴바 버튼 클릭 시 contenteditable의 선택 영역(커서/드래그)이 소실되어 색상이 적용되지 않던 문제 수정. `selectionchange` 이벤트로 선택 영역을 지속 저장하고, 색상 적용 직전 복원. Bold·Italic 버튼도 동일하게 처리
- **툴바 버튼 mousedown 시 선택 영역 소실 방지** — 툴바의 B·I·Color 버튼에 `mousedown` 이벤트에서 `preventDefault()`를 적용하여 포커스가 이동하지 않도록 수정
- **편집 종료 후 텍스트 편집 가능 상태 유지 버그 수정** — 편집 모드 종료 시 `contenteditable="true"` 속성이 제거되지 않아 뷰 모드에서도 텍스트가 편집되던 버그 수정. 편집 종료 시 `.page` 내 모든 `contenteditable="true"` 요소에서 속성 일괄 제거
- **URL 링크 클릭 불가 버그 수정** — 편집 종료 후에도 `contenteditable`이 유지되어 `<a>` 링크 클릭 시 내비게이션이 되지 않던 문제 수정. 위 contenteditable 제거 수정으로 함께 해결
- **신규 객체(태그·칩) 추가 시 삭제 버튼 × 가 텍스트로 인식되는 버그 수정** — 새 태그/칩 생성 후 자동 선택 시 삭제 버튼 `×`까지 선택에 포함되어 첫 입력 시 삭제 버튼이 사라지던 문제 수정. 선택 시 텍스트 노드만 선택하도록 `selectTextOnly()` 헬퍼 추가. 태그·칩 삭제 버튼에 `contenteditable="false"` 및 `user-select: none` 적용하여 편집 중 문자 단위 삭제 방지

#### 신규 기능
- **성과 칩(outcome-chip) 드래그 순서 변경** — 경력 블록 하단 성과 칩을 마우스로 드래그하여 좌우 순서 변경 가능. 태그바와 동일한 드래그 UX 적용
- **URL 자동 링크 변환** — 텍스트 필드에 `https://` URL 입력 후 필드를 벗어나면 자동으로 클릭 가능한 `<a>` 링크로 변환. 편집 모드 종료 후 실제 클릭 가능. 연락처, 포지셔닝, 불릿, 업무 스타일 등 본문 영역 전체 지원

#### UI 개선
- **Core 태그 색상 변경 팝업 위치 수정** — 우클릭 색상 변경 팝업이 툴팁(hover card)과 동일 위치에 겹치던 문제 수정. 팝업을 태그 하단 → 태그 상단으로 이동하여 충돌 해소
- **경력 블록 삭제 버튼 위치 수정** — `✕ 경력` / `✕ Phase` 삭제 버튼이 텍스트(재직기간, Phase기간)와 겹쳐 시인성이 낮던 문제 수정. 헤더에 오른쪽 여백을 확보하여 버튼과 텍스트가 겹치지 않도록 레이아웃 개선
